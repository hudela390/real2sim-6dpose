Real-to-Sim 6-DoF Object Pose

Depth-sensor-free object-pose tracking through grasp, transport and release — recovering the full 6-DoF trajectory of a manipulated object from a single egocentric RGB video, so teleoperated robot demonstrations can be replayed in simulation.

Team 22

ETH Zürich · 3D Vision course project

Overview

We recover the full per-frame 6-DoF trajectory of a manipulated object from a single egocentric RGB stream — no depth sensor — so that teleoperated robot demonstrations can be transferred into a physics simulator. This is hard: the objects are near-symmetric (a rubber duck, a circular pan), so appearance alone leaves rotation ambiguous, and the gripper heavily but intermittently occludes the object throughout grasp, transport and release. We therefore anchor one high-quality pose and propagate it with a model-based tracker, correcting it with the robot's joint log and the object mask rather than trusting any single per-frame estimate.

  • Input: egocentric RGB @ 50 fps + a teleoperation log (h5) of robot & gripper joints; known CAD mesh.
  • No metric depth sensor — depth is estimated monocularly and scale-calibrated against the object silhouette.
  • Output: a per-frame 4×4 object trajectory (green CAD box + axes overlaid below) that a simulator can replay.

Method

A model-based base track, anchored once, then corrected per phase from whichever signal is most trustworthy. The rollout is partitioned at the ungrasp into an in-hand part and a drop-to-rest part.

1

Front end

Per-frame object mask (SAM2 / SAM3) + a scale-calibrated monocular-depth stream. No depth sensor.

2

Base + anchor

RGBTrack base track, anchored by a frame-0 GoTrack rotation refinement to fix the ambiguous start.

3

In-hand

While gripped: position from the mask, depth from the gripper forward kinematics (h5), rotation from GoTrack as a cleaned noisy teacher.

4

Drop → rest

Once released and unoccluded, full-6-DoF GoTrack tracks the fall; outliers dropped, lightly smoothed. The static rest is its tail.

5

Compose

Concatenate the two parts into the per-frame 4×4 trajectory consumed by the simulation replay.
